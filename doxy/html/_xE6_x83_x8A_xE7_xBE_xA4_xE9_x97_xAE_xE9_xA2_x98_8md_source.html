109 于是于是我们知道 pthread_mutex_t 的底层调用了 futex 并唤醒了一个线程！ 但是 futex 是如何唤醒线程的呢？继续查看其具体实现（kernel/futex.c）：

114 static int futex_wake(u32 __user *uaddr, int fshared, int nr_wake, u32 bitset)
115 {
116  struct futex_hash_bucket *hb;
117  struct futex_q *this, *next;
118  struct plist_head *head;
119  union futex_key key = FUTEX_KEY_INIT;
120  int ret;
121 
122  ...
123  //根据uaddr的值填充&key的内容
124  ret = get_futex_key(uaddr, fshared, &key, VERIFY_READ);
125  if (unlikely(ret != 0))
126  goto out;
127  //根据&key获得对应uaddr所在的futex_hash_bucket
128  hb = hash_futex(&key);
129  //对该hb加自旋锁
130  spin_lock(&hb->lock);
131  head = &hb->chain;
132  //遍历该hb的链表，注意链表中存储的节点是plist_node类型，而而这里的this却是futex_q类型，这种类型转换是通过c中的container_of机制实现的
133  plist_for_each_entry_safe(this, next, head, list) {
134  if (match_futex (&this->key, &key)) {
135  ...
136  //唤醒对应进程
137  wake_futex(this);
138  if (++ret >= nr_wake)
139  break;
140  }
141  }
142  //释放自旋锁
143  spin_unlock(&hb->lock);
144  put_futex_key(fshared, &key);
145 out:
146  return ret;
147 }
148 ```
149 
150 我们看到 futex 用了一个自选锁保护了一个 futex_hash_bucket 链表，里面存放等待唤醒的线程，在唤醒过程中，遍历链表逐一唤醒，一旦唤醒数量达到需求的唤醒量就 break。
